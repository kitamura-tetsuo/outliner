[Issues]
During code review and exploration, a few minor code health issues were discovered:
1. Svelte a11y warnings (`a11y_click_events_have_key_events`, `a11y_no_static_element_interactions`) were bypassed using `svelte-ignore` comments in `OutlinerItem.svelte` rather than being addressed properly using `role`, `tabindex`, and keyboard interaction handlers.
2. Standard `console.log`, `console.warn`, and `console.error` methods were still being utilized in some parts of the client logic (`src/lib/yjs/testHelpers.ts`, `src/schema/app-schema.ts`), despite the project's strategy to use a structured `logger` via `src/lib/logger.ts`.

[Changes]
1. `OutlinerItem.svelte`: Removed `svelte-ignore` comments for a11y warnings. Added `role="treeitem"`, `aria-selected`, and `tabindex="-1"` attributes to the main item container. Added `role="button"` and `tabindex="-1"` to the content display `div`. Attached `onkeydown` handlers that listen for `Enter` and `Space` key presses to trigger the corresponding `handleClick` / `handleContentClick` logic, properly fulfilling standard accessibility patterns.
2. `app-schema.ts` and `testHelpers.ts`: Replaced raw `console.*` invocations with standard `logger.info`, `logger.warn`, and `logger.error` using string interpolation to gracefully handle dynamic error messages, resolving type issues while preserving log structure.

[Verification Results]
Verified that `npm run check` passes correctly with 0 errors and 0 warnings, confirming the a11y issues were correctly remediated. Verified `npm run lint` correctly parses the syntax and completes.
